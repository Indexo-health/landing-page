# Bevel.health 参考总结与修改意见（供审核）

> 目的：吸收 Bevel 的优点，不照抄视觉；用于优化 Indexo Health 的 landing page。

## Bevel 的关键优点（可借鉴）
- **Hero 结构清晰**：一句强主张 + 一段简短说明 + 单一主 CTA；信息密度低但记忆点强。
- **大图主导氛围**：Hero 以全宽大图塑造情绪与质感，文本区独立且留白充足。
- **强视觉节奏**：大图 → 分节内容 → 产品演示 → 生活场景 → 口碑 → CTA 的叙事路径清晰。
- **模块化卡片一致性**：卡片圆角/边框/阴影统一；视觉“家族感”强。
- **信任结构完善**：有 App/产品背书、用户评价、FAQ、合规/法律入口。
- **导航和页脚信息完整**：顶部导航简洁，页脚分列清晰（Company / Product / Resources / Legal + 社媒）。
- **动效克制但有质感**：大图、手机 mock、轻微视差、进入动画增加高级感但不喧宾夺主。

## 可迁移的方向（不照抄）
- **更强 Hero 叙事**：让首页“情绪 + 价值 + 视觉”同时成立，减少冗长解释。
- **更明确的分节节奏**：每节给出“标题/一句话/要点/图”的固定节奏，避免信息混杂。
- **统一卡片语言**：统一圆角/阴影/边框/留白比例，减少模块“风格漂移”。
- **信任与合规结构前置**：把“风险说明/合规文案”放在“Trust”附近，避免页尾的生硬警告。
- **CTA 专注**：每个大节只保留一个主要 CTA，减少多按钮干扰。
- **视觉层次强化**：大标题、轻正文、短要点，减少长段文字。
- **界面证据强化**：用 3–5 张关键界面/报告图，把抽象承诺变成“看得见的结果”。
- **价值四宫格结构**：用 2x2 模块拆解复杂价值（每块只讲一个结果）。
- **AI 服务人格化**：用“对话式预览”展示系统如何解释与提醒，而不是只说“AI”。

## 对 Indexo 页面可执行的修改意见（建议优先级）
### P0（结构与信息）
1. **Hero 重新分层**：
   - 主标题 + 一句价值解释 + 单一 CTA。
   - 右侧/背景主视觉完整呈现（避免遮挡）。
2. **“问题/解决/价值”三段彻底区分**：
   - Problem 只讲痛点；Solution 讲 Pad + Ring；Value 讲“确定的安心/家属通知/晨间简报”。
3. **统一文案密度**：
   - 每个区块不超过 2 行说明；把长文本拆成 2–3 条短 bullet。
4. **四宫格价值拆解**：
   - 睡眠质量 / 呼吸与缺氧 / 心脏与恢复 / 家属安心，形成 2x2 结构。

### P1（视觉与品牌）
1. **统一卡片规范**：
   - 统一圆角（例如 24px/32px）、阴影强度、边框色；所有卡片沿用同一“组件语言”。
2. **配色更有层次**：
   - 以品牌主色 + 中性色层级做“标题/正文/辅助”三档；减少大面积纯色块。
3. **更有节奏的留白**：
   - 标题区与内容区间距加大，区块之间留白更明显。
4. **截图呈现一致**：
   - 统一手机框/透视角/背景色，避免“拼贴感”。

### P2（信任与转化）
1. **引入“背书/引用/评语”模块**：
   - 2–3 条短句式评价或“适合谁”短卡。
2. **页脚完善**：
   - Company / Product / Resources / Legal + 联系方式（已加入结构）。
3. **合规说明专业化**：
   - 改为正式医疗免责声明，不使用“placeholder”。
4. **重复 CTA 但样式一致**：
   - Hero / 中段 / 页尾三处 CTA 保持同一视觉体系。
